# redis-cli -h [접근 서버 IP] -p [레디스 실행 프로세스 포트] -a [비밀번호]: 외부에서 접속
	- 외부의 접속(다른 호스트의 클라이언트를 redis 연결)을 위해 conf 파일에서 protected-mode no 로 바꿈.

- 캐시 삭제 시 java event listener 설정. 
	redis.conf 에서 notify-keyspace-events "Ex" 로 설정. 재시작
	- 캐시 ttl이 0 이 될 때 발행되는 것이 아닌 redis 서버가 캐시를 삭제할 떄 발행됨.
	- 근데 안해도("" 로 설정해도) 가능함 => 라고는 하는데 설정안하니까 정상작동이 안되서 "Ex" 설정 넣음.
	- Ex 설정은 성능적 오버헤드가 발생할수 도 있음
