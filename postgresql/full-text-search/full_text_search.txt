Full Text Search(전문검색): postgresql nlp 자연어 기반 구문 검색

- GIN(Generic Inverted Index) Index: Full text 검색 속도를 높이는데 사용 가능한 index.
	GIN index는 역 인덱스 구조를 사용.
	키워드나 토큰과 같은 작은 단위의 단어를 인덱싱.
	값이 아닌 키를 저장하고 검색함.
	B-tree index는 짧은 문자역 혹은 숫자타입의 컬럼에서 효과적이지만, Text 타입처럼 대량의 문자열의 경우 GIN index가 효과적.

- Full Text Search Index 전처리
	1) 문서를 토큰으로 파싱-숫자, 단어, 특수문자 이메일 주소 등 다양한 클래스의 토큰을 식별해서 이 클래스들이 다르게 처리되도록 함.
		문장: [“AI is transforming the world”] =>  토큰: [“AI”, “is”, “transforming”, “the”, “world”]
	2) 토큰을 Lexeme(어휘)으로 변환. 같은 단어의 다른 형태가 비슷하게 만들어지도록 정규화, 이 단계에서 불용어(stopword) 제거
		정규화: “AI” → “ai”, “transforming” → “transform” => 불용어 제거: [“is”, “the”] 제거 => 렉셈: [“ai”, “transform”, “world”]
	3) 정규화된 렉셈을 위치정보와 함께 저장
		정렬된 렉셈 리스트: [“ai”, “transform”, “world”] => 위치 정보: {“ai”: [0], “transform”: [1], “world”: [2]}
		
	
- Full Text Search Date Type
	1) tsvector: 효율적인 텍스트 검색에 최적화된 형식으로 전처리된 문서를 반환(렉셈과 위치정보를 나타낸다)
		
	
	2) tsquery: 검색쿼리를 나타내고, 전처리된 렉셈을 반환. Lexeme(어휘)의 정렬된 목록과 문서 내에서의 위치와 가중치가 포함. 
		연산자 사용해서 검색쿼리에 사용(&, | ,!, ", :, - 등)
		우선순위(높은순부터): !, <->, &, |
		
	3) @@ (매치 연산자): 문서의 텍스트(tsvector)와 검색쿼리에 지정된 용어(tsquery)사이의 유사성을 평가. boolean 타입 반환
	
		
		
1. column 생성 type: tsvector
2. index 생성
	CREATE INDEX [index 명] ON [table] USING gin ([column]);
	
예문 
SELECT to_tsvector('english','hello world! The pineapple is very delicious. apple is my favorite food. jump. jumped.') @@ to_tsquery('english','jumping');

함수:
	- to_tsvector()
	- to_tsquery()
	- plainto_tsquery()
	- phraseto_tsquery()
	- websearch_to_tsquery()
	- ts_rank_cd()
