1. 가장 권장하는 방법: up 명령어 재실행 
	이미지가 바뀌었을 때 가장 안전하고 확실하게 컨테이너를 갱신하는 방법은 다시 빌드하고 up 명령어를 사용하는 것입니다. 
	bash
	# 1. 원격 저장소에서 최신 이미지 가져오기
	docker compose pull

	# 2. 이미지 기반으로 컨테이너 다시 빌드 및 실행 (이미지 변경 시)
	docker compose up -d --build
	
2. 특정 서비스만 이미지 갱신하여 재시작
	특정 서비스(web이라고 가정)만 새로 불러온 이미지로 업데이트하고 싶을 때 사용합니다.
	bash
	docker compose pull {서비스명}
	docker compose up -d --no-deps {서비스명}
	
	
3. local image -> docker tar -> -> docker image
	1. image->tar
		docker save -o all.tar 이미지1:태그 이미지2:태그
	2. tar->image
		docker load -i 파일명.tar
