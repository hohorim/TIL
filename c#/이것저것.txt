## visual studio 
	코드 편집 및 탐색
	•	코드 자동 완성: Ctrl + Space
	•	정의로 이동: F12
	•	구현으로 이동: Ctrl + F12
	•	참조 찾기: Shift + F12
	•	빠른 액션 및 리팩터링: Ctrl + .
	•	줄 복사: Ctrl + C (줄 선택 없이 커서만 있으면 전체 줄 복사)
	•	줄 삭제: Ctrl + Shift + L
	•	줄 이동: Alt + ↑/↓
	파일 및 심볼 찾기
	•	모든 파일/기호 빠르게 찾기: Ctrl + T 또는 Ctrl + ,
	•	파일 내 검색: Ctrl + F
	•	솔루션 전체 검색: Ctrl + Shift + F
	디버깅
	•	디버깅 시작/계속: F5
	•	디버깅 중단: Shift + F5
	•	한 줄씩 실행(Step Over): F10
	•	한 줄씩 진입(Step Into): F11
	•	한 줄씩 나가기(Step Out): Shift + F11
	•	중단점 토글: F9
	•	다음 중단점까지 실행: Ctrl + F5
	기타
	•	솔루션 탐색기 열기: Ctrl + Alt + L
	•	출력 창 열기: Ctrl + Alt + O
	•	단일 파일 닫기: Ctrl + F4
	•	모든 문서 닫기: Ctrl + Shift + F4

	•	사용할 때는 PascalCase(프로퍼티)

## Namespace
	C#에서 클래스, 인터페이스, 함수 등을 논리적으로 묶는 이름 공간
	여러 파일에서 같은 네임스페이스 사용해도 상관없음.
		단, 같은 네임스페이스에 같은 이름의 클래스/인터페이스/구조체는 하나만 존재해야함.
		서로 다른 네임스페이스에서 동일한 이름의 클래스/인터페이스/구조체 가능.

## 어셈블리(assembly) 
	.NET 에 컴파일된 결과물(DDL, EXE)
	여러 네임스페이스와 클래스를 하나의 어셈블리(ex: MyLibrary.ddl) 에 담을 수 있다.
	어셉블리에 정의되어 있다 == 해당 타입이 어떠한 ddl(또는 exe) 파일 안에 들어있다
	
## using(import)
	같은 네임스페이스에 있으면 using 정의없이도 같은 네임스페이스 안에서 서로 사용할 수 있다.
	프로젝트가 ddl 을 참조하고 있으면 그 참조 네임스페이스만 using 구문 추가하면 그 타입을 별도의 using 없이 쓸 수 있다.
	
	
## CommunityToolkit.Mvvm 도구 
	MVVM 패턴을 쉽게 구현할 수 있도록 다양한 유틸리티 클래스 제공
	
	- 주요 클래스
		- ObservableObject: ViewModel이나 Model에서 속성 변경 알림(INotifyPropertyChanged)을 쉽게 구현할 수 있도록 하는 기본 클래스입니다.
		- ObservableProperty (Attribute): 필드에 붙이면 자동으로 속성(Property)과 변경 알림 코드를 생성해줍니다.
			변경알림 받을 메서드는 partial void On[PropertyName]Changed 패턴을 쓰면 값이 바뀔 때마다 해당 메서드가 자동 호출
			또, 위 필드를 붙이면, 필드(멤버변수)가 되어 클래스 내에서 전역적으로 사용 가능하고, 자동으로 get,set 을 생성해줌
			실제 값이 변경되고 On[Property]Changed 이벤트 직후(Changed), 직전(Changing).
			
		- RelayCommand / RelayCommand<T>: 버튼 클릭 등 UI 명령을 ViewModel 메서드와 연결할 수 있게 해주는 커맨드 클래스입니다.
			xaml 에서 Binding 을 {함수명}+Command 형태로 선언하면 
			Viewmodel 의 cs 에서 [RelayCommand] 가 붙으면서 동일한 {함수명}이 매핑되어 이벤트가 실행된다.
			단, cs에서 함수명 접미사로 "Async", "Command"는 존재한다면 자동 제외된다.
			
		
		- AsyncRelayCommand / AsyncRelayCommand<T>: 비동기 명령(예: 서버 통신 등)을 쉽게 처리할 수 있는 커맨드 클래스입니다.
		- ObservableRecipient: 메시지(Messenger) 기능과 속성 변경 알림을 함께 제공하는 ViewModel용 기본 클래스입니다.
		- WeakReferenceMessenger: ViewModel, View 등에서 메시지를 약한 참조로 주고받을 수 있게 해주는 메시징 시스템입니다.
		- INotifyPropertyChanged, INotifyPropertyChanging: 속성 변경 전/후 알림을 위한 인터페이스 및 구현체를 제공합니다.
		
## View의 xaml 과 ViewModel 의 cs 연결은 
	xaml 코드비하인드에 DataContext 에 매핑할 cs 를 인스턴스로 생성하는데, 이 DataContext가 연결고리 역할을 하여 자동으로 동기화함.
	일관된 파일명은 개발자 편의와 관례일 뿐임.
	
## class의 parial
	C#에서 partial 키워드는 하나의 클래스(또는 struct, interface)를 여러 파일에 나눠서 정의할 수 있게 해주는 기능입니다.
	단, 동일한 클래스명/네임스페이스이여야 하고, partial 키워드가 모두 붙어 있어야 가능.
	partial은 여러 파일에 나눠 작성된 클래스/구조체/인터페이스를 컴파일 시 하나로 합쳐주는 C# 문법
	
## ObservableCollection 타입
	ObservableCollection 로 선언된 변수가 변경되면 자동으로 UI에 반영해줌
	List 는 단순 컬렉션이기 때문에 자동반영되지 않음
	
## Task 반환값
	- Task 반환: 비동기 작업, 반환값 없음(void와 유사)
	- Task<T> 반환: 비동기 작업, T 타입의 결과값 반환
